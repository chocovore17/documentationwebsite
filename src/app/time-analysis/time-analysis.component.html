Time analysis

Initiation intervals were set manually for each task and the maximum execution time was measured by taking 32 separate readings for each task and the average found.

Initiation Intervals and Maximum Execution Time

# Task Initiation interval Maximum Execution Time Critical Instant Analysis
1 DisplayUpdateTask 100ms 101.08ms 1
2 MsgOutTask 10ms 9.48ms 10
3 scanKeysTask 10ms 53.60ms 10
4 DecodeMsgTask 5ms 4.67ms 20

Critical Instant Analysis of Rate Monotonic Scheduler

For a single CPU, the tasks have fixed execution times. The fixed initiation interval for each task is also its deadline. There are no dependencies or switching overheads. The task priority is fixed and the shortest initiation has the highest priority. 0.0 The worst case is if every task is initiated at the same time. If so the latency is:

Ln=100/10 x 9.48 + 100/10 x 53.60 + 100/5 x 4.67 + 100/100 x 101.08 Ln=824.2 ms
